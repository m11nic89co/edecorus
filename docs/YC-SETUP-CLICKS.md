# Yandex Cloud: куда кликать (15 минут)

Откройте в браузере: **https://console.yandex.cloud/**  
Войдите под своим Яндекс-ID (телефон / почта). Карта нужна для аккаунта; в free tier списания обычно 0 ₽.

После каждого блока можно проверить URL из подсказки.

---

## Блок A. Каталог (папка в облаке)

1. Если спрашивают «Создать каталог» → имя: `edecorus` → **Создать**.
2. Убедитесь, что вверху выбран каталог **edecorus** (или ваш рабочий).

---

## Блок B. Бакет `edecorus.ru`

1. Меню слева → **Object Storage** (Объектное хранилище).
2. Кнопка **Создать бакет**.
3. Заполните:
   - **Имя:** `edecorus.ru` (точно как домен).
   - **Класс хранилища:** Стандартное.
   - **Доступ:** ограничить публичный доступ — **выключено** (нужен публичный доступ к объектам).
   - **Шифрование:** **не включать** (иначе хостинг сайта недоступен).
4. **Создать бакет**.

### Публичное чтение

5. Откройте бакет `edecorus.ru` → вкладка **Настройки** (или **Обзор** → настройки доступа).
6. **Чтение объектов** → **Для всех**.
7. **Чтение списка объектов** → **Для всех**.
8. **Сохранить**.

### Хостинг сайта

9. Вкладка **Веб-сайт** (или **Настройки** → **Веб-сайт**).
10. Режим: **Хостинг** (не «Переадресация всех запросов»).
11. **Главная страница:** `index.html`
12. **Страница ошибки:** `404.html`
13. **Сохранить**.
14. Скопируйте **ссылку на сайт** вида:  
    `https://edecorus.ru.website.yandexcloud.net`  
    Откройте в браузере — пока будет пусто или 404 (файлы зальём дальше).

---

## Блок C. Ключи для деплоя

1. Меню → **Identity and Access Management** (IAM) / **Управление доступом**.
2. **Сервисные аккаунты** → **Создать сервисный аккаунт**.
3. Имя: `edecorus-deploy` → **Создать**.
4. Откройте аккаунт → вкладка **Роли** → **Назначить роли**.
5. Роль: **`storage.editor`** (или `storage.admin`) → каталог `edecorus` → **Сохранить**.
6. Вкладка **Статические ключи доступа** → **Создать ключ** → тип **Создать статический ключ доступа**.
7. **Скопируйте и сохраните** (показывают один раз):
   - **Идентификатор ключа** (Access Key ID)
   - **Секретный ключ** (Secret Access Key)

## Блок D. Заливка сайта с ПК

В PowerShell (папка проекта):

```powershell
pip install awscli
aws configure --profile yc
# Access Key ID: вставить из блока C
# Secret Access Key: вставить из блока C
# region: ru-central1
# output: json

cd C:\Users\AM\Projects\Y\edecorus.ru
copy scripts\yandex-deploy-config.example.ps1 scripts\yandex-deploy-config.local.ps1
.\scripts\deploy-yandex-s3.ps1
```

Снова откройте `https://edecorus.ru.website.yandexcloud.net` — должен открыться сайт Ёдекор.

## Блок E. GitHub (автодеплой)

1. https://github.com/m11nic89co/edecorus/settings/secrets/actions
2. **New repository secret:**
   - `YC_SA_ACCESS_KEY_ID` = идентификатор ключа
   - `YC_SA_SECRET_ACCESS_KEY` = секретный ключ
3. **Actions** → workflow **Deploy to Yandex Cloud** → **Run workflow** (или push в `main`).

---

## Блок F. Домен edecorus.ru (когда тест OK)

1. У регистратора домена (где куплен edecorus.ru) → DNS.
2. Удалить A/CNAME на **GitHub Pages**.
3. Добавить **CNAME** для `@` или `www`:
   - `edecorus.ru` → `edecorus.ru.website.yandexcloud.net`
4. В Yandex: бакет → **Веб-сайт** → привязать **собственный домен** `edecorus.ru` (если есть мастер в консоли).
5. GitHub → **Settings → Pages** → убрать custom domain `edecorus.ru`.

Готово.
